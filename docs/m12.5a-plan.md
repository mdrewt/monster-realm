# M12.5a Plan: fuse offspring monster_pub dual-write fix

## Bug
evolution.rs:359-361: pub_from_monster called BEFORE insert, so pub row gets monster_id=0.
- Production: `offspring_monster.monster_id == 0` (auto_inc default before insert)
- `let offspring_pub = pub_from_monster(&offspring_monster)` → pub has id=0
- `ctx.db.monster().insert(offspring_monster)` → SpacetimeDB assigns real id (return discarded)
- `ctx.db.monster_pub().insert(offspring_pub)` → pub lands with id=0
- Second fuse aborts on PK-0 collision → fusion permanently broken after one use

## Seam divergence
fuse_seam in evolution_tests.rs pre-allocates id with alloc_monster_id() → tests never see id=0 bug.

## Fix (12.5a-1)
Capture insert return value (mirrors movement.rs:104-105):
  `let inserted = ctx.db.monster().insert(offspring_monster);`
  `ctx.db.monster_pub().insert(pub_from_monster(&inserted));`

## Seam alignment (12.5a-2)
- Change TestEvolutionDb::insert_monster to auto-assign id (when 0) and return Monster
- Remove alloc_monster_id() pre-allocation from fuse_seam; set monster_id: 0
- Update fuse_seam insert block to use return value pattern
