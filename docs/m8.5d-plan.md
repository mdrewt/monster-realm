## EARS criteria (all four in this one coherent slice)
1. `just lint` runs `cargo fmt --all --check` AND `biome check .` in addition to clippy; CI
   toolchain components include `rustfmt`.
2. Every GitHub Action under `.github/` SHA-pinned (`@<40-hex> # vX`).
3. `client/package.json` declares `"engines": { "node": ">=24.13.1 <25" }`; `log` promoted to
   root `[workspace.dependencies]` and consumed via `log.workspace = true`.
4. Devcontainer `postCreateCommand` (`just setup`) succeeds — pinned Rust 1.96.0
   (+wasm32/clippy/rustfmt), pinned Node 24.13.1, wasm-pack 0.15.0.

## Anti-patterns / landmines
- `cargo fmt --workspace` → rejected; use `--all`.
- dtolnay master-SHA → CI red ("'toolchain' is a required input"); use the @stable-branch SHA.
- biome gating generated `module_bindings` / dev-automation `.claude/` → false gate.
- global-disabling `noNonNullAssertion` under a "generated noise" pretense → dishonest/toothless;
  scope off for tests only.
- adding the biome/fmt gate BEFORE the autofix commit → reds CI for everyone.
- `new RegExp` in eval code → Semgrep `detect-non-literal-regexp` (bitten 3×) reds remote CI.
- second Rust-channel SSOT in devcontainer → drift vs rust-toolchain.toml.
- claiming EARS-4 runtime-proven when only a static check ran → state the human smoke-test.
