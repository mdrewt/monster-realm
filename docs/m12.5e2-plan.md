# M12.5e2 Build Plan: Practice-battle XP multiplier (0.1Ă—)

**Branch:** `feat/m12.5e2-practice-xp`  
**ADR:** 0078 (`docs/adr/0078-practice-xp-multiplier.md`)  
**Touches:** `game-core/src/combat/xp.rs`, `game-core/src/combat/mod.rs`,
`game-core/src/lib.rs`, `server-module/src/battle.rs`, `server-module/src/battle_tests.rs`

---

## Scope

Drew's decision (2026-07-03): self-/PvP-battle XP is "practice XP" at 0.1Ă—
multiplier. Wild battles (WILD_IDENTITY) keep full XP.

EARS criteria:
1. WHEN a player wins a battle where `opponent_identity != WILD_IDENTITY`,
   THE XP earned SHALL be `floor(base_xp / 10)` (practice multiplier, min 0).
2. WHEN a player wins a battle where `opponent_identity == WILD_IDENTITY`,
   THE XP earned SHALL be the full `battle_xp_reward(...)` value (no change).
3. THE practice XP rule SHALL be a pure function in `game-core` (SSOT, ADR-0003).

Out of scope: NPC sentinels (future), schema changes, rate-limiting.

## Architecture

### Functional core (game-core)

**New function** in `game-core/src/combat/xp.rs`:
```rust
/// Apply the practice-battle XP multiplier (0.1Ă—, floor) to a base XP reward.
///
/// Called when the opponent is NOT WILD_IDENTITY (e.g. a self-battle or
/// future PvP). Wild encounters pass the base reward directly without calling
/// this. Result can be 0 (e.g. base 1â€“9 XP â†’ 0 practice XP).
///
/// ADR-0078: SSOT for the 0.1Ă— practice penalty rule.
pub fn practice_xp(base_xp: Xp) -> Xp {
    Xp::new(base_xp.value() / 10)
}
```

**Export chain:**
- `game-core/src/combat/mod.rs` line 39: add `practice_xp` to `pub use xp::{...}`
- `game-core/src/lib.rs` line 30: add `practice_xp` to `pub use combat::{...}`

### Imperative shell (server-module)

In `write_back_battle_results` (`server-module/src/battle.rs`, ~line 721),
immediately before `apply_xp_gain`:

**Before:**
```rust

```

**After:**

## Test plan

### Unit tests in `game-core/src/combat/xp.rs`

All in the existing `#[cfg(test)] mod tests` block:

1. **`practice_xp_known_answer_floor`** â€” `practice_xp(Xp::new(64))` == `Xp::new(6)`
   (floor(64/10)=6; kills wrong-operator impls)

2. **`practice_xp_floor_truncates_to_zero`** â€” `practice_xp(Xp::new(9))` == `Xp::new(0)`
   (kills a min-1 impl; confirms floor behavior at the 0-boundary)

3. **`practice_xp_full_base`** â€” `practice_xp(Xp::new(100))` == `Xp::new(10)`
   (kills an off-by-one impl using `/10` vs `*0.1` rounding divergence)

4. **Property `prop_practice_xp_never_exceeds_base`** â€” `practice_xp(x).value() <= x.value()`
   for all `x` in 0..u32::MAX/2 (kills any impl that inflates XP)

5. **`practice_xp_zero_input`** â€” `practice_xp(Xp::new(0))` == `Xp::new(0)` (edge case)

### Structural source-guard tests in `server-module/src/battle_tests.rs`

Using the existing `extract_fn_body` / `strip_rust_comments` pattern:

6. **`write_back_battle_results_calls_practice_xp`** â€” verifies `practice_xp` appears
   in the `write_back_battle_results` body (kills inline `/ 10` replacing the call)

7. **`write_back_battle_results_practice_xp_gated_on_wild_identity`** â€” verifies
   `WILD_IDENTITY` appears in the body AND that `practice_xp` appears AFTER the
   `WILD_IDENTITY` check (kills an impl that applies the multiplier to all battles)

**Proof-of-teeth requirement:** reverting the `practice_xp` call in `battle.rs`
must turn tests 6 and 7 RED; adding `/ 10` inline without calling `practice_xp`
must turn test 6 RED; calling `practice_xp` unconditionally (no WILD_IDENTITY
guard) must turn test 7 RED.

## Anti-patterns to avoid

- Do NOT inline `/ 10` in `battle.rs` â€” that breaks the SSOT (ADR-0003)
- Do NOT set minimum to 1 â€” the spec allows 0 practice XP for small rewards
- Do NOT change the `battle_xp_reward` signature â€” callers outside battle.rs
  would be affected; the multiplier is applied post-compute
- Do NOT add a `is_practice` parameter to `battle_xp_reward` â€” that would leak
  provenance into the pure combat rule; the provenance check is a server concern
