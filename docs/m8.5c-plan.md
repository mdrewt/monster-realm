## 1. Dual-write proof-of-teeth — NEW `evals/monster-dual-write.eval.mjs` (+ register in `evals/run.mjs`)
Invariant: every monster-mutating function mirrors `monster`→`monster_pub` (same op), so the public
projection can never silently diverge. Field parity is ALREADY covered by Rust unit tests
`monster_from_instance_flattens_correctly` (server lib.rs ~2143) + `pub_from_monster_omits_hidden_fields`
(~2170) — the eval does NOT duplicate that; it enforces the MIRROR.

- **Syntax coverage (CRITICAL):** monster writes use TWO syntaxes — inserts `ctx.db.monster().insert(` and
  updates `ctx.db.monster().monster_id().update(` (and potential `.delete(`). A naive scan for
  `monster().update(` misses ALL 5 update sites (lib.rs 1042/1085/1723/1748/1849). Match both forms.
- **Scope by FUNCTION, not line-adjacency:** the multi-line write_back flows span dozens of lines; an
  adjacency window is brittle. Split the file into fn bodies; for any fn containing a monster mutation,
  require a corresponding `monster_pub` mutation of the same kind in that fn. Inserts must build the pub via
  `pub_from_monster(`. (Update sites build a `pub_row` upstream — require the `monster_pub().monster_id().update(`
  mirror, not necessarily `pub_from_monster` on the same line.)
- **Proof-of-teeth (must bite):** (a) a fn string doing `ctx.db.monster().monster_id().update(m);` with NO
  `monster_pub` mirror → pass:false; (b) a fn doing `ctx.db.monster().insert(row)` whose monster_pub mirror
  is hand-rolled (no `pub_from_monster(`) → pass:false. A green run on real source is expected (mirrors are
  currently all paired).
- **NO dynamic `new RegExp`** — literal regex / `String.includes`/`indexOf` only (Semgrep ReDoS bit 3×).

## 3. bindings-drift in `ci`
Today: `just eval` runs bindings-drift in the `ci` job but it SKIPS (no spacetime CLI → pass:true). The e2e
job has the CLI but runs `just e2e` (Playwright), not `just eval` → drift is NEVER actually caught.
- **Install + pin spacetime CLI in the `ci` job** (mirror e2e job ci.yml ~96-107) so `just eval`→bindings-drift
  runs the real `spacetime generate` + diff. Backstop-only (no install) is WRONG: it would red EVERY ci run.
- **Harden `evals/bindings-drift.eval.mjs`** no-CLI branch: if `process.env.CI` set AND committed bindings
  exist AND CLI absent → pass:false (guards a regressed install step); else SKIP (local CLI-less stays green).
- **Proof-of-teeth:** existing `diffDirs` teeth stay; add a check that the fail-loud branch returns pass:false
  under CI+bindings+no-CLI (stub `hasCli`).
- Cost (module build in ci) accepted per brief & recorded in ADR-0050. (Coordinate `.github/` with M8.5d → serial.)
