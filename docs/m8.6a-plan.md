## Design (post plan-review: planner + reviewer + red-team, lenses synthesized)
- Add `BattleSide::set_active(&mut self, idx: u32) -> Result<(), SwapError>` to `types.rs`.
  Rejects **OutOfBounds** (`idx as usize >= team.len()`, covers empty) then **Fainted**
  (`team[idx].is_fainted()`) — **bounds before fainted** (order is load-bearing; pinned by test).
  `SwapError { OutOfBounds, Fainted }`, derives `Debug,Clone,Copy,PartialEq,Eq`, no serde/SpacetimeType,
  not `#[non_exhaustive]`. Doc-comment names the contract + ADR-0053.
- Reroute all 6 raw `active =` writes in `resolve.rs` through `set_active` (zero raw assignments remain):
  - `resolve_turn` Swap A/B (`:168`,`:175`): `if set_active(idx).is_ok() { push Switch }` else no-op, turn proceeds.
  - `resolve_player_swap` (`:322`,`:323`): on `Err` return early — empty events, **no enemy turn**, no mutation.
  - auto-switch (`:111`,`:112`): infallible idx from `next_conscious_index`; call unconditionally, consume `Result`,
    `debug_assert!` the success (NOT around the side-effecting call). Comment the invariant.
- `pub active` stays; doc-comment names `set_active` as sole sanctioned mutator (privatization PARKED, see ADR-0053).

## Decisions from plan review (all in ADR-0053)
1. **Drop `AlreadyActive`** — swap-to-self is neither panic nor corruption; it's server policy (kept as
   defense-in-depth in `swap_active`, out of touch-set). Core rejects exactly spec §3: OOB + Fainted. (YAGNI.)
2. **Proof-of-teeth load-bearing assertions** — fainted case must assert `active == pre_active` + no `Switch` +
   `!active_monster().is_fainted()` (NOT just "no panic"; field is pub → a reverted raw setter won't panic on a
   fainted in-bounds index). OOB case asserts no panic + active unchanged.
3. **resolve_turn (no-op-proceed) vs resolve_player_swap (full-abort) asymmetry** — named/justified; M16 owns PvP policy.
4. **Signatures frozen** — server-module calls the resolvers (out of touch-set); rejection = absence of Switch event.

## Anti-patterns to forbid
clamp-instead-of-reject · change resolver signatures · widen touch-set (lib.rs/monster/redteam_new_findings.rs) ·
toothless proof (fainted case without the `active`-unchanged assertion) · side-effect inside `debug_assert!` ·
`#[allow]` to silence clippy · any `unwrap`/`expect`/`panic` in `set_active` · weaken/delete `m7b_5`/`m7b_5b`.
