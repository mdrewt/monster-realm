## Exported-function seam (tester writes teeth against these; implementer builds red→green)

### A — `evals/spacetime-type-snapshot.eval.mjs` (16.5e-1)
- `export function checkAppendOnly(prevBaseline, newBaseline) -> string[]` — pure. For every type in BOTH maps: enum → `new.variants.slice(0, prev.length)` must deep-equal `prev.variants` (tail-append only; mid-insert/reorder/removal/rename-at-any-prefix-position flagged); struct → same rule over `[name,type]` field pairs. Kind flip enum↔struct → flagged. Type removed vs prev → flagged. New types → allowed. `[]` = clean.
- **Layering (review M-1):** `checkTypeDrift` (parsed SOURCE vs committed baseline, exact-match) stays byte-for-byte unchanged. `checkAppendOnly` is an ADDITIONAL, separate comparison with different inputs: prev-committed baseline (git) vs the working-tree baseline JSON file. A legit tail-append updates source+baseline together: exact-match green (source==new baseline), append-only green (new ⊇-prefix prev). No interaction.
- Internal `readPrevBaseline()` — git I/O isolated from the pure fn: (1) `git merge-base HEAD origin/master` → `git show <sha>:evals/baselines/spacetime-types.json`; (2) fallback `git show origin/master:…`; (3) `{source:'none', data:null}`. **Self-identical rule (review B-1):** if the primary comparison is vacuous because prev deep-equals the working baseline (the master-push CI case: merge-base==HEAD, clean tree — and any clean-checkout-on-master run), additionally validate the LAST transition: `git show HEAD~1:…` vs `git show HEAD:…`. This closes the direct-push/self-compare hole; on a repo with one commit, HEAD~1 fails → D2 skip. Wire into default export after the existing BattleOutcome tooth; on `source:'none'` apply D2 (fail-open-loud).
- Inline self-proof tooth (A-8, pinned per review M-4): doctored **mid-insert on BattleOutcome** — real baseline `[Ongoing, SideAWins, SideBWins, Fled]` vs doctored `[Ongoing, Doctored, SideAWins, SideBWins, Fled]` must be flagged by `checkAppendOnly`, else RED immediately. (Distinct from the existing variant-ADD tooth, which exercises `checkTypeDrift`.)

### B — `evals/trade-escrow-guards.eval.mjs` (16.5e-2)
- `export function orderAndFilterRustEntries(entries) -> string[]` — pure (review m-1: minimal seam, no readFileFn injection): filter `.rs`, **exclude `*_tests.rs`**, **sort**. `readAllRustSources(dir)` maps over it with `readFileSync`.
- `export function stripRustStrings(src) -> string` — the same `/"(?:[^"\\]|\\.)*"/g` → `""` strip `bodyHasGuard` already uses.
- `extractFunctionBody` gains a whole-source string-strip (comments first, then strings) so a string-literal `pub fn <name>(` cannot become the extraction anchor. REAL-tree justification (review B-2): `economy_tests.rs` contains `"pub fn sell("` in an assert string today — current correctness is file-order luck. `bodyHasGuard`'s per-body strip and ALL RT-SEC-02/02b teeth stay untouched.
- Documented residuals (review M-3): raw strings `r#"…"#` not stripped (pre-existing); a `*/` inside a normal string can corrupt comment-then-string stripping (comment regex fires first) — accepted, named in ADR-0116; no hand-rolled Rust lexer.

### C — `evals/bsatn-compat-smoke.eval.mjs` (16.5e-3)
- `export function parseContentTableStructs(schemaSrc)` — `#[spacetimedb::table(name = X` attr → struct→table map + `pub <field>: Option<…>` fields.
- `export function parseSyncedTables(contentSrc)` — write targets across **all of content.rs** (D4 — captures `seed_heal_locations_from`). Needle must tolerate multi-line chains + intermediate index accessors (review m-4): content.rs has `ctx.db\n .shop_item_row()\n .shop_item_id()\n .delete(…)` and `ctx.db.heal_location_row().location_id().update(row)`. Use ONE literal regex with bounded lookahead, e.g. `/ctx\s*\.\s*db\s*\.\s*(\w+)\s*\(\)[\s\S]{0,120}?\.\s*(update|insert|delete)\s*\(/g` (literal — Semgrep-safe; bounded quantifier — no ReDoS).
- `export function checkAdditiveColumnCoupling(…) -> string[]` — every Option field on a synced content table needs (a) Option in schema.rs AND (b) **a field-assignment `<field>:` inside a `<StructName> {` row-literal block in content.rs** (review M-2: clear-and-reinsert tables — shop_item_row/type_relation_row/fusion — have NO update branch by design; requiring `update(` would false-positive future Option columns there. The field-assignment check encodes "the re-seed path writes this column" for BOTH the upsert shape (one row literal feeds update+insert) and clear-and-reinsert, and catches the `..existing`-spread refactor omission directly). Row-literal blocks are extracted by locating `<StructName> {` occurrences in stripped content.rs and brace-walking (the existing extractor pattern). **Vacuity guard:** zero Option fields discovered → loud FAIL. **Anchors:** `ability`, `train_stat`, `cure_status`, `cost_item_id` must be among discovered fields. Residual (documented): two synced tables sharing an identically-named Option field could mask each other only if their row literals use the same struct name — impossible (struct names are unique) — so masking requires same-struct duplication; accept and note.

## ADR-0116 decisions

- **D1 (resolves spec D-16.5-3):** enforce per-PR inside the existing evals under `just eval`/`just ci`; no justfile edit. On master-push CI merge-base==HEAD → 16.5e-1 vacuously green there (its teeth are per-PR — intended).
- **D2:** git/prev-baseline unavailable → **fail-open with loud detail** (`append-only check SKIPPED: source=none`). Bounded: the unchanged exact-match gate still fails any source-vs-baseline drift; the **`ci` job's eval step** (the one that runs `just eval`) always checks out with fetch-depth:0 (the e2e job is shallow but runs only ci-gate-wiring), so the skip can only happen off-CI. Fail-closed would red offline local runs for a per-PR check.
- **D3:** prev-baseline source = `git show $(git merge-base HEAD origin/master):evals/baselines/spacetime-types.json`, fallback origin/master tip, then skip. Merge-base avoids behind-master false positives. **Self-identical amendment (B-1):** when prev deep-equals the working baseline, validate the last transition `HEAD~1→HEAD` instead — closes the master-push/direct-push vacuous-compare hole. `execFileSync` const-args only.
- **D4:** 16.5e-3 write/assignment search scoped to **all of content.rs** (not the fn body) — `heal_location_row.cost_item_id` is seeded by a helper outside `sync_content_inner`; fn-body scope would false-positive. `encounter`'s row literal lives in marshal.rs but its `.update(` call site is in content.rs — captured; `encounter` has no Option columns, and D4's field-assignment rule (M-2 amendment) is what a future one would be held to. Coupling realizes the spec's "update-branch write" as field-assignment-in-row-literal because row literals feed both branches and clear-and-reinsert tables have no update branch by design.
- **D5:** in-place type edit of an existing field at a prefix position IS flagged by `checkAppendOnly` (prefix compare over `[name,type]` pairs) — a wire break; keep flagged.

## Risks

- **R1** comment-vs-string strip ordering (strip comments then strings; reuse the existing conservative regex verbatim; raw strings `r#"…"#` stay a documented residual; NO hand-rolled Rust lexer).
- **R2** behind-master false positive → merge-base (D3).
- **R3** git unavailable → D2.
- **R4** vacuous parser rot → C-3 vacuity guard + C-4 anchors (m16.5a lesson).
- **R5** over-broad scope flagging `monster.evolves_to` → synced-tables-only coupling (C-5).
- **R6** Semgrep detect-non-literal-regexp → zero `new RegExp`; literal `/regex/` + indexOf; `execFileSync('git',[…])`.
- **R7** RED phase: teeth must CALL not-yet-existing exports (clean assertion RED via run.mjs synthetic fail), not top-level-crash the module import. **Placement (review M-5):** new teeth go AFTER the existing tooth blocks (existing teeth keep executing first during RED); the temporary short-circuit of later real-file checks within the one RED eval is accepted — loud, temporary, and the other 60 evals are unaffected. Final green ordering keeps teeth before real-file checks per file convention.
- **R8** ADR digest gate → `Subsystems: ci-gates`, run `just adr-digest`.
- **R9** pre-existing `parseSpacetimeTypes` multi-line-variant limit — inherited, not worsened.
