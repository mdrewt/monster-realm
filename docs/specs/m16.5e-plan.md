## Exported-function seam (tester writes teeth against these; implementer builds red‚Üígreen)

### A ‚Äî `evals/spacetime-type-snapshot.eval.mjs` (16.5e-1)
- `export function checkAppendOnly(prevBaseline, newBaseline) -> string[]` ‚Äî pure. For every type in BOTH maps: enum ‚Üí `new.variants.slice(0, prev.length)` must deep-equal `prev.variants` (tail-append only; mid-insert/reorder/removal/rename-at-any-prefix-position flagged); struct ‚Üí same rule over `[name,type]` field pairs. Kind flip enum‚Üîstruct ‚Üí flagged. Type removed vs prev ‚Üí flagged. New types ‚Üí allowed. `[]` = clean. **Never throws (red-team F1/F2):** kind mismatch is checked-and-flagged BEFORE any array access; malformed entries (non-array `variants`/`fields`) produce a diagnostic flag string, not a TypeError ‚Äî a throw would fail-closed via run.mjs but with an opaque message.
- **Layering (review M-1):** `checkTypeDrift` (parsed SOURCE vs committed baseline, exact-match) stays byte-for-byte unchanged. `checkAppendOnly` is an ADDITIONAL, separate comparison with different inputs: prev-committed baseline (git) vs the working-tree baseline JSON file. A legit tail-append updates source+baseline together: exact-match green (source==new baseline), append-only green (new ‚äá-prefix prev). No interaction.
- Internal `readPrevBaseline()` ‚Äî git I/O isolated from the pure fn: (1) `git merge-base HEAD origin/master` ‚Üí `git show <sha>:evals/baselines/spacetime-types.json`; (2) fallback `git show origin/master:‚Ä¶`; (3) `{source:'none', data:null}`. **Self-identical rule (review B-1):** if the primary comparison is vacuous because prev deep-equals the working baseline (the master-push CI case: merge-base==HEAD, clean tree ‚Äî and any clean-checkout-on-master run), additionally validate the LAST transition: `git show HEAD~1:‚Ä¶` vs `git show HEAD:‚Ä¶`. This closes the direct-push/self-compare hole; on a repo with one commit, HEAD~1 fails ‚Üí D2 skip. Wire into default export after the existing BattleOutcome tooth; on `source:'none'` apply D2 (fail-open-loud).
- Inline self-proof tooth (A-8, pinned per review M-4): doctored **mid-insert on BattleOutcome** ‚Äî real baseline `[Ongoing, SideAWins, SideBWins, Fled]` vs doctored `[Ongoing, Doctored, SideAWins, SideBWins, Fled]` must be flagged by `checkAppendOnly`, else RED immediately. (Distinct from the existing variant-ADD tooth, which exercises `checkTypeDrift`.)

### C ‚Äî `evals/bsatn-compat-smoke.eval.mjs` (16.5e-3)
- `export function parseContentTableStructs(schemaSrc)` ‚Äî `#[spacetimedb::table(name = X` attr ‚Üí struct‚Üítable map + `pub <field>: Option<‚Ä¶>` fields.
- `export function parseSyncedTables(contentSrc)` ‚Äî write targets across **all of content.rs** (D4 ‚Äî captures `seed_heal_locations_from`). Needle must tolerate multi-line chains + intermediate index accessors (review m-4): content.rs has `ctx.db\n .shop_item_row()\n .shop_item_id()\n .delete(‚Ä¶)` and `ctx.db.heal_location_row().location_id().update(row)`. Use ONE literal regex with bounded lookahead, e.g. `/ctx\s*\.\s*db\s*\.\s*(\w+)\s*\(\)[\s\S]{0,120}?\.\s*(update|insert|delete)\s*\(/g` (literal ‚Äî Semgrep-safe; bounded quantifier ‚Äî no ReDoS).
- `export function checkAdditiveColumnCoupling(‚Ä¶) -> string[]` ‚Äî every Option field on a synced content table needs (a) Option in schema.rs AND (b) **a field-assignment `<field>:` inside a `<StructName> {` row-literal block in content.rs** (review M-2: clear-and-reinsert tables ‚Äî shop_item_row/type_relation_row/fusion ‚Äî have NO update branch by design; requiring `update(` would false-positive future Option columns there. The field-assignment check encodes "the re-seed path writes this column" for BOTH the upsert shape (one row literal feeds update+insert) and clear-and-reinsert, and catches the `..existing`-spread refactor omission directly). Row-literal blocks are extracted by locating `<StructName> {` occurrences in stripped content.rs and brace-walking (the existing extractor pattern). **Word-boundary rule:** the `<field>:` search must reject substring false-matches (`ability:` inside `stability:`, `ability_id:`) ‚Äî use indexOf plus a manual preceding-char check (`[A-Za-z0-9_]` before the match ‚áí not a field label); NO dynamic RegExp. Schema Option-field regex must tolerate whitespace/newline after the colon (`/pub\s+(\w+)\s*:\s*Option</g`, literal). Qualified `core::option::Option<‚Ä¶>` is not used in schema.rs ‚Äî unhandled, documented residual (anchors defend). **Vacuity guard:** zero Option fields discovered ‚Üí loud FAIL. **Anchors:** `ability`, `train_stat`, `cure_status`, `cost_item_id` must be among discovered fields. Residual (documented): two synced tables sharing an identically-named Option field could mask each other only if their row literals use the same struct name ‚Äî impossible (struct names are unique) ‚Äî so masking requires same-struct duplication; accept and note.

## ADR-0116 decisions

- **D1 (resolves spec D-16.5-3):** enforce per-PR inside the existing evals under `just eval`/`just ci`; no justfile edit. On master-push CI merge-base==HEAD ‚Üí 16.5e-1 vacuously green there (its teeth are per-PR ‚Äî intended).
- **D2:** git/prev-baseline unavailable ‚Üí **fail-open with loud detail** (`append-only check SKIPPED: source=none`). Bounded: the unchanged exact-match gate still fails any source-vs-baseline drift; the **`ci` job's eval step** (the one that runs `just eval`) always checks out with fetch-depth:0 (the e2e job is shallow but runs only ci-gate-wiring), so the skip can only happen off-CI. Fail-closed would red offline local runs for a per-PR check.
- **D3:** prev-baseline source = `git show $(git merge-base HEAD origin/master):evals/baselines/spacetime-types.json`, fallback origin/master tip, then skip. Merge-base avoids behind-master false positives. **Self-identical amendment (B-1):** when prev deep-equals the working baseline, validate the last transition `HEAD~1‚ÜíHEAD` instead ‚Äî closes the master-push/direct-push vacuous-compare hole. `execFileSync` const-args only.
- **D4:** 16.5e-3 write/assignment search scoped to **all of content.rs** (not the fn body) ‚Äî `heal_location_row.cost_item_id` is seeded by a helper outside `sync_content_inner`; fn-body scope would false-positive. `encounter`'s row literal lives in marshal.rs but its `.update(` call site is in content.rs ‚Äî captured; `encounter` has no Option columns, and D4's field-assignment rule (M-2 amendment) is what a future one would be held to. Coupling realizes the spec's "update-branch write" as field-assignment-in-row-literal because row literals feed both branches and clear-and-reinsert tables have no update branch by design.
- **D5:** in-place type edit of an existing field at a prefix position IS flagged by `checkAppendOnly` (prefix compare over `[name,type]` pairs) ‚Äî a wire break; keep flagged.
