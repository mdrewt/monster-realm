# m17.5d build plan — leaderboard profile.name passive mirror (EARS 17.5d-1/2)

Planner output (fable, 2026-07-18), reviewed by reviewer + red-team lenses. Spec: M17.5 §17.5d.
Declared touches: `server-module/src/ranking.rs` + `server-module/src/ranking_tests.rs` (+ standard doc outputs, ADR-0125).

## Problem (verified @ a0d5743)

`get_or_init_profile` (ranking.rs:27-47) sets `name` only in the insert (None) arm; `apply_pvp_rating`
(ranking.rs:58-89) updates via `Profile { rating, .., ..winner }` spreads which keep the old name.
`join_game` (movement.rs:44) recreates the `player` row with a fresh `validate_name`'d name each
session; `player` rows are deleted on disconnect. No path updates `profile.name` → the public
leaderboard permanently shows the first-ever rated name.

## Design (passive mirror ONLY — no set_profile_name reducer; D-17.5-C stays parked)

1. **Pure core** (executed-testable, m17.5a precedent):
   `fn refresh_profile_name(profile: Profile, live_name: Option<String>) -> Profile`
   — `Some(n)` → `Profile { name: n, ..profile }`; `None` → unchanged (disconnect-forfeit keeps
   last-known name). No ctx, no write, no empty-string special-casing (`validate_name` guarantees
   `player.name` non-empty at the write boundary — stated assumption, documented in ADR).
2. **Ctx helper** dedup'ing the player lookup both arms use:
   `fn live_player_name(ctx: &ReducerContext, identity: Identity) -> Option<String>`
   — inline chained `ctx.db.player().identity().find(identity).map(|p| p.name)` (split-binding ban).
3. **Wiring:** `get_or_init_profile`'s `Some` arm becomes
   `Some(existing) => refresh_profile_name(existing, live_player_name(ctx, identity))`
   (in-memory only, NO extra DB write); the `None` arm reuses `live_player_name(...).unwrap_or_default()`.
   `apply_pvp_rating`'s existing `..winner`/`..loser` spreads persist the refreshed name for BOTH
   parties on every rated write — no change to `apply_pvp_rating`'s body.

Behavior matrix: rename→rated game = new name persisted; player row absent = keep existing name;
unrated battle = no write (rename surfaces on NEXT rated game, per spec); first rating = insert arm
seeds from live row (unchanged).

## Byte-pins that MUST survive (existing teeth in pvp_tests.rs + evals/ranking-security.eval.mjs)

1. No literal `1000` outside comments in ranking.rs (rl7-iii)
2. Exactly ONE `compute_rating_update(` call (rl7-ii)
3. `SideAWins => (battle.player_identity, battle.opponent_identity)` / `SideBWins => (battle.opponent_identity, battle.player_identity)` byte-stable (RT-M17-01)
4. `.wins.saturating_add(1)` / `.losses.saturating_add(1)` / `is_ranked_pvp(` in apply_pvp_rating body (rl5-v, rl7-iv)
5. `compute_rating_update(` before first `profile().identity().update(`/`profile().insert(` in the body (rl7-viii)
6. No `#[spacetimedb::reducer]` / `reducer as ` in ranking.rs (rl7-i)
7. NEVER `= ctx.db.profile()` (repo-wide split-binding ban); `ctx.db.profile()` only in ranking.rs (eval A2)
8. `mod ranking_tests` decl + fn name `get_or_init_profile` preserved

## Proof-of-teeth map (mutation → killing test)

| Mutation | Killed by |
|---|---|
| Some-arm refresh deleted / result discarded / `None` passed literally | `d1_get_or_init_some_arm_refreshes_name` (composed-call needle) |
| refresh ignores live_name | `d1_refresh_replaces_name_when_live_present` |
| refresh uses `unwrap_or_default()` (clobbers with `""` on None) | `d1_refresh_keeps_name_when_absent` |
| refresh mutates rating/wins/losses | `d1_refresh_replaces_name_when_live_present` (stats-unchanged asserts) |
| rename not persisted through spread | `d2_rename_then_rated_surfaces_new_name` + pins #3/#4 (spreads untouched) |

Targeted mutation run: `cargo mutants -p monster-realm-module -f src/ranking.rs` — expect 0 missed.

## Anti-patterns (named)

1. Writing the profile row inside the `Some` arm (extra write per lookup; changes write-count shape).
2. Adding any name-edit reducer (parked D-17.5-C; ADR-0119 D6 module-write-only).
3. `unwrap_or_default()` in the refresh path (empty-string clobber).
4. Split-bindings `= ctx.db.player()`/`= ctx.db.profile()`.
5. Touching pvp_tests.rs (helpers get a local copy in ranking_tests.rs).
6. Over-pinning: no duplicate needles for what rl7 already pins.

## Review dispositions (plan fan)

- R1 (red-team, ADOPTED): T2's needle must pin the composed call shape as the Some arm's returned
  expression, not the bare fn-name presence — closes discard-result / literal-None gate-holes.
- R2 (reviewer, ADOPTED): pure fns can be private (`fn`, not `pub(crate)`) — ranking_tests is a
  child module via `#[path]`, so `super::` reaches private items; keep the surface minimal.
- Solo-workflow recommendation accepted (no debate/compete — no genuine design ambiguity).
