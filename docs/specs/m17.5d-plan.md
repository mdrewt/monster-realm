## Problem (verified @ a0d5743)

`get_or_init_profile` (ranking.rs:27-47) sets `name` only in the insert (None) arm; `apply_pvp_rating`
(ranking.rs:58-89) updates via `Profile { rating, .., ..winner }` spreads which keep the old name.
`join_game` (movement.rs:44) recreates the `player` row with a fresh `validate_name`'d name each
session; `player` rows are deleted on disconnect (AFTER forfeit_on_disconnect settles ratings — the
forfeiter's row is still live when rating applies; a simultaneous-disconnect race can make the
counterparty's row absent, which the None branch must tolerate). No path updates `profile.name` →
the public leaderboard permanently shows the first-ever rated name.

## Design (passive mirror ONLY — no set_profile_name reducer; D-17.5-C stays parked)

1. **Pure core** (executed-testable, m17.5a precedent), private `fn`:
   `fn refresh_profile_name(profile: Profile, live_name: Option<String>) -> Profile`
   — `Some(n)` → `Profile { name: n, ..profile }`; `None` → unchanged (disconnect keeps last-known
   name). No ctx, no write, no empty-string special-casing (`validate_name` guarantees `player.name`
   non-empty at the write boundary — stated assumption, documented in ADR D2).
2. **Ctx helper**, private `fn`, dedup'ing the player lookup both arms use:
   `fn live_player_name(ctx: &ReducerContext, identity: Identity) -> Option<String>`
   — body is EXACTLY the inline chain `ctx.db.player().identity().find(identity).map(|p| p.name)`
   (no split-binding, no `.unwrap()` — the `.map` form is what makes the disconnect-race None safe).
3. **Wiring:** `get_or_init_profile`'s `Some` arm becomes
   `Some(existing) => refresh_profile_name(existing, live_player_name(ctx, identity))`
   (in-memory only, NO DB write); the `None` arm's name becomes
   `live_player_name(ctx, identity).unwrap_or_default()` (same helper, both arms).
   `apply_pvp_rating`'s body is UNCHANGED — its existing `..winner`/`..loser` spreads persist the
   refreshed name for BOTH parties on every rated write.

Behavior matrix: rename→rated game = new name persisted (either role); player row absent = keep
existing name; unrated battle = no write (rename surfaces on NEXT rated game, per spec); first
rating = insert arm seeds from live row (unchanged).

## Test suite (T1 executed + T2 source-scan, all in ranking_tests.rs)

## Anti-patterns (named)

1. Writing the profile row inside the `Some` arm (extra write per lookup) — now GATED by update-count pin.
2. Adding any name-edit reducer (parked D-17.5-C; ADR-0119 D6 module-write-only; rl7-i scan).
3. `unwrap_or_default()`/`unwrap()` in the refresh/lookup path (empty-clobber / panic) — gated.
4. Split-bindings `= ctx.db.player()`/`= ctx.db.profile()` — now gated in-file by T2.
5. Touching pvp_tests.rs (helpers get local copies in ranking_tests.rs).
6. Verbatim (non-concat!) needles that self-match ranking_tests.rs (reviewer W-4).
