## B) f-3 approach — DECIDED

**Eval-diff is the load-bearing gate; runtime check is a fail-soft log-never-throw net. Do NOT replace the `{ tag: string }` structural interfaces.**

1. REJECT type-replacement arm: rowConvert.ts:1-5/:37-39 documents deliberate SDK-decoupling (store never imports SDK; tests build plain objects). A type-only import of generated unions also would NOT catch the bug — structural widening keeps tsc green (the spec's own evidence says so).
2. `evals/sdk-enum-exhaustiveness.eval.mjs`: parse `module_bindings/types.ts` enum-variant lists (`__t.enum("Name", {...})`), diff against a client-side exhaustive-handling registry exported from rowConvert.ts. Server-added variant → regen widens types.ts → eval RED. Mirrors spacetime-type-snapshot pattern; complementary to it (server BSATN append-only vs client exhaustive handling).
3. Runtime `narrowTag` helper: on unknown tag, log (console.warn-based reporter) and return the raw value — NEVER throw (flushBatch has NO per-listener isolation; a throw kills the whole batch). Centralizes the :525 cast into one audited site.

### T3 — f-2 e2e: client/e2e/trade-interlock.spec.ts (NEW — named `trade-interlock` so it sorts AFTER golden.spec/recruit.spec, eliminating the presenceCount===2 cross-contamination hazard; red-team F7 / reviewer W-3)
- **Paired positive-control design (red-team BLOCKERs F1/F2 — the load-bearing anti-false-green mechanism):**
  1. Two chromium.launch(); gameReady both (fresh identities → fresh starters at full HP, which also moots the KO-race F10).
  2. **POSITIVE CONTROL (pre-battle):** B `__mrTrade.proposeTrade` with the EXACT final argument shape — initiatorMonsterIds=[B's starter], counterpartyMonsterIds=[A's starter], zero items/currency (non-empty both sides so validate_proposal TR-1 cannot fire; monsters are owned, so ownership guards pass) → assert the offer row APPEARS (poll allTradeOffers) → B cancelTrade → assert row gone (so the active-offer guard cannot confound the second call).

### T4 — f-3 runtime check: client/src/net/rowConvert.ts + rowConvert.test.ts
- Add `narrowTag<T extends string>(raw, known: readonly T[], enumName): T` — returns raw as T when known; on unknown: reportUnknownTag(enumName, raw) via console.warn, return raw via ONE centralized cast (fail-soft, log-NEVER-throw — flushBatch has no per-listener isolation).

This two-gate ratchet is the call-site coupling; the eval additionally pins that :525 references `HANDLED_ENUM_VARIANTS.TradeStatus` textually so the coupling cannot be severed. For registry entries WITHOUT a narrowTag site (string-typed stores), the eval alone forces author awareness — accepted limitation, recorded in ADR-0127.
- Tests (RED until narrowTag exists): known tag passes through typed; unknown tag returns raw AND logs (spy); tradeOfferRowToStore with {tag:'FutureVariant'} does NOT throw.
