## Context and problem statement

M8.5-hardening Â§3 requires `just lint` to run `biome check .` in addition to clippy, so
the pre-commit hook and `just ci` actually gate formatting (EARS-1). A `biome.json`
(schema 2.5.1, formatter on, linter `recommended` on) was committed earlier but **never
wired into any recipe** â€” so biome's config has never been enforced. Turning it on as-is
surfaced **90 errors + 167 warnings across 96 files**. Three problems make a naive "add
`biome check .` to `just lint`" wrong:

1. **biome is not installed anywhere** (not in any `package.json`, not on PATH), and there
   is **no repo-root `package.json`** â€” so `npx biome` from the repo root would net-fetch
   nondeterministically (or fail offline).
2. **biome would gate code it must not own.** `files.includes` was `["**", "!**/*.json",
   "!**/*.jsonc"]`. That includes `client/src/module_bindings/**` (regenerated by
   `spacetime generate` â€” gating it is a false SSOT that breaks on every regen) and the
   tracked `.claude/hooks/*.mjs` local-dev automation (biome only honors `.gitignore`, and
   `.claude/` is deliberately NOT gitignored; Semgrep already excludes it via
   `--exclude '.claude'`).
3. **`recommended` includes high-churn, non-autofixable, behavior-adjacent style rules.**
   The dominant one is `lint/style/noNonNullAssertion` â€” **117 occurrences, of which 114
   are in hand-written test files** (`*.test.ts`, idiomatic `Map.get()!` / `team[i]!`
   assertions) and **3 are in production `client/src/ui/battleModel.ts`** (`team[active]!`).
   Forcing all 117 to be rewritten is a real null-safety refactor â€” behavior-adjacent and
   out of the spirit of a build/CI hygiene slice whose stated intent is to *gate formatting*.

## Decision

1. **Install biome as a `client/` devDependency** (`@biomejs/biome` pinned to `2.5.x`,
   matching the `biome.json` schema) and invoke it from the repo root via the deterministic
   path `client/node_modules/.bin/biome check .` in `just lint`. No root `package.json` is
   created (YAGNI).
2. **Exclude generated/vendored/dev-automation** in `biome.json` `files.includes`:
   `!client/src/module_bindings`, `!.claude`, `!client/dist`. `node_modules` stays
   covered by `vcs.useIgnoreFile`. (NB: biome 2.5.1 treats a bare directory negation and its
   `/**` form as equivalent and **canonicalizes `!dir/**` back to `!dir` on every
   `biome check --write`** â€” the bare form is the stable/idempotent one and is what is
   committed. Verified empirically: `biome check client/src/module_bindings/` â†’ "Checked 0
   files".)
3. **Enforce formatting + the auto-fixable lint rules now.** A one-time
   `biome check --write` pass auto-fixes `format`, `noUnusedImports`, `useImportType`,
   `useTemplate`, `useOptionalChain`, `useLiteralKeys` across the owned source. These rules
   stay **enforced**.
4. **Disable `noNonNullAssertion` for test files only**, via a `biome.json` `overrides`
   entry scoped to `**/*.test.ts`. The rule remains **enforced on all production source** â€”
   a future `!` in non-test code is still caught. The 3 production sites in
   `battleModel.ts` are handled in-slice (fixed to a checked access if behavior-preserving,
   else covered by a narrow, documented override) â€” NOT by a global disable.
5. **Defer the test-file `noNonNullAssertion` cleanup** (re-enable the rule for tests by
   replacing the 114 `!` assertions with proper null-handling) to a dedicated lint-debt
   slice. It is behavior-adjacent test work, not build hygiene.

## Considered alternatives

- **Global-disable `noNonNullAssertion`.** Rejected: it would silence the rule for
  production code too (where null-safety matters), and the original plan's premise that the
  debt was "generated-code noise" is **false** â€” the violations are in hand-written code.
  A global disable under that pretense is a toothless/dishonest gate.
- **Fix all 117 sites in this slice.** Rejected: ~114 are in tests and a few are
  behavior-adjacent (`team[active]!`); bundling a null-safety refactor into a tooling slice
  violates the slice's intent and adds review risk. Deferred to a named slice instead.
- **Run `biome check .` against the whole tree without exclusions.** Rejected: gates
  generated bindings (breaks on regen) and local dev automation.
- **`npx @biomejs/biome` from repo root.** Rejected: no root `node_modules`; nondeterministic
  net fetch / offline failure.

## Consequences

- `biome check .` is green-and-meaningful: it gates formatting + the autofixed correctness/
  style rules over all owned TS/JS, with `noNonNullAssertion` enforced on production and
  deferred on tests (auditable via the `overrides` block + this ADR).
- The autofix pass and the `cargo fmt --all` reformat are each committed as their own step so
  the gate-wiring diff stays reviewable (spec Â§6).
- The `overrides` / `includes` edits are the SSOT for "what biome gates"; the deferred
  test-file cleanup is tracked as a named follow-up slice.
