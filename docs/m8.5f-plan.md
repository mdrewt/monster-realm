## Sub-increments

### A — NET-1 desync (→ ADR-0052)  [predictor.ts, main.ts, server-module/lib.rs comment, predictor.test.ts]
**Fix = enforce the move-queue-cap invariant on the client `#queue` at both mutation
points** (single-sourced cap, mirrors the server's reject-when-full/keep-head):
1. `Predictor.enqueue(input): IntentToSend | undefined` — when `#queue.length >=
   #queueCap`: do not push, do not `#record` (no seq consumed), return `undefined`.
   `setMove`/`clearQueue` unchanged (cap-safe, length ≤ 1).
2. `reconcile` clamps after replay: `this.#queue = q.slice(0, this.#queueCap)`.
3. `main.ts sendIntent`: `const intent = predictor.enqueue(input); if (intent ===
   undefined) return; conn.conn.reducers.enqueueMove(...)`.
4. Server: **no structural change** — add a comment at the `authorize_move` ack site
   documenting that the over-cap `Err` rolls back the accept-time ack (so "ack only on
   successful enqueue" holds by SpacetimeDB transaction semantics). The structural ack
   split is considered-and-rejected (ADR-0052; YAGNI + missed-ack risk; rollback +
   client cap-invariant already guarantee it).

**Proof-of-teeth (predictor.test.ts, small-cap predictor cap=2):**
- `burst > cap → reconcile`: enqueue East ×5 → calls 3–5 return `undefined`,
  `queueDepth`/`pendingCount` never exceed cap; drain; reconcile against the
  server-accepted queue → predicted == authoritative (no over-prediction). BITES the
  unbounded enqueue.
- `authQueue surprise → reconcile clamps`: fill `#queue` to cap with unacked enqueues,
  reconcile against `authQueue=[Z]` acking nothing → `#queue.length == cap` (the excess
  is clamped, predicted advances ≤ cap). BITES a missing reconcile clamp (would
  over-predict by 1).

### B — lazy `#lastDrainAt`  [predictor.ts, predictor.test.ts]
`#lastDrainAt: number | undefined = undefined`; in `drain`,
`snapped = this.#lastDrainAt !== undefined && now - this.#lastDrainAt >
SNAP_GAP_STEPS * this.#stepMs` then set `#lastDrainAt = now`.
**Honest note (reviewer B-2 + verified):** the spurious first-drain snap is already
masked publicly (reconcile's internal `drain` sets `#lastDrainAt`, and `drain`
early-returns while `predicted === undefined`), so this is **correctness-by-
construction** — it makes reconcile's internal first-drain honest and guards a future
reconcile refactor; it is *not* a live observable bite. Tests: keep the existing snapped
cases green (predictor.test.ts:251–265) + a regression-guard documenting the invariant.

cap/exports); predictor depending on the SDK; over-refactoring the shell; weakening an
existing snapped test; dynamic `new RegExp` in any eval (ReDoS — literal/`String`
matching only); adding a cap check to setMove/clearQueue.
