## The 4 items + their proof-of-teeth

### 1. Schema-snapshot generalization (`evals/battle-schema-snapshot.eval.mjs`; ADR-0006 enforcement)
Generalize from battle-only to **all 15** `#[spacetimedb::table]` structs. Parser must capture **column-set + PK + declared field types**; the eval **exact-matches** each table vs a consolidated baseline.
- **Decisions:** D1 exact-match (any drift add/remove/PK/type fails → reviewable baseline update, per §6); D2 single `evals/baselines/table-schemas.json` keyed `table → {pk, columns:{name:type}}`; D3 keep the eval filename (avoid ARCHITECTURE/ADR rename churn); D4 delete `battle-columns.json`. Baseline generated by a **one-shot** reusing the eval's exported parser, **committed as its own step** (no persistent generator script — keeps surface minimal).
- **Parser requirements (from review):** capture types (`pub <name>: <type>,`), detect PK via the `#[primary_key]` **attribute** (not name presence), **strip comments first** (reuse `stripRustComments` idiom) so commented-out fields aren't phantom-captured, key on the `#[spacetimedb::table(` attribute so `EncounterEntryRow` (a `#[derive(SpacetimeType)]` struct) is **excluded**. Static regex literals only (no dynamic `new RegExp` — Semgrep gate).
- **Teeth that MUST bite (on a NON-`battle` table — `inventory`/`encounter`):** column drop · PK change · declared-type change · additive column (proves exact-match) · green-path match passes · `EncounterEntryRow` absent from the parsed map.

### 2. Zoned-schema broadening (`evals/zoned-schema.eval.mjs`; ADR-0007)
Broaden from tile-bearing-only to flag **any** table declaring a `zone_id`/`map_id` field that is neither `#[primary_key]` nor `#[index(btree)]`.
- **D5 scheduler carve-out (VALIDATED by red-team vs ADR-0007):** exempt tables whose `#[spacetimedb::table(...)]` attribute contains `scheduled(`. Real `movement_tick_schedule.zone_id` is dispatch payload (no `.zone_id()` accessor, not subscribed) → not a world table → not required to be indexed. **Implementation constraint:** `parseTables` must expose the **attribute** string (`m[0]`), and the carve-out checks that, NOT the struct body.
- **Teeth:** keep the existing zoneless-spatial `ghost` tooth (FLAG) · add encounter-shaped tooth (zone_id as PK → PASS) · add bare-`zone_id` tooth (FLAG) · add `scheduled(`-table tooth (unindexed zone_id → PASS). Real source must stay green (character indexed, zone_def/encounter PK, movement_tick_schedule carved out).

### 3. recruit-reducer-security strengthening (`evals/recruit-reducer-security.eval.mjs`)
Strengthen `checkOwnershipGuard` + `checkWildBattleGuard` from substring-presence to **rejecting-comparison**.
- **Ownership (CRITICAL — must pass real `me`-alias code):** resolve `let <alias> = ctx.sender;`, then require `player_identity != <alias>` OR `player_identity != ctx.sender` (whitespace-collapsed `indexOf`) followed by `Err(`/`return Err` within a bounded window after the `!=`. Real code: `let me = ctx.sender; … if battle.player_identity != me { …; return Err(e); }` (lib.rs:1927/1936).
- **Wild:** require a `battle_wild(...)` **lookup** whose not-found path rejects (`None => { … return Err }` / `ok_or` / `?`) within a window — not mere `battle_wild(` presence (the GC `.delete()` also contains it).
- **New bad fixtures that MUST bite:** `BAD_OWNERSHIP_NO_REJECTION` (mentions `player_identity` + `ctx.sender` but no `!=`+Err — the old substring check wrongly passes it) · `BAD_WILD_NO_REJECTION` (`battle_wild(` only in GC `.delete()`, no lookup+reject). Existing GOOD fixture + the **real** `attempt_recruit` must still PASS.

### 4. IV-inversion test fix (`game-core/src/combat/redteam_m8d_tests.rs`)

Pick known `iv = 15`; compute `public_stat_hp` via `derive_stats` ONCE (input construction, not a self-oracle — the assertion is on the *search result*).
