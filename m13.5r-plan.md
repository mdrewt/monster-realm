Due to Rust's operator precedence (&&  > ||), `a || b || c` with first `||` → `&&` becomes

all others same. Then: mutant → `(false && true) || false || ... = false` → no update (wrong).
Original → `false || true || false ... = true` → update. Test catches the difference.

Fix: 5 tests that each change EXACTLY ONE FIELD and assert an Update is planned:
1. Only home_x changes → Update (kills mutant 492)
